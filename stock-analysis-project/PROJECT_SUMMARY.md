## 核心功能实现

### 1. 中短线稳健型交易策略分析系统 (`trading_strategy.py`)
**严格遵循用户提供的策略框架：**

#### 标准化选股流程（三步筛选法）
1. **趋势筛选**：检查均线多头排列，股价在关键均线上方
2. **量价验证**：分析成交量变化，确认资金流入
3. **基本面筛查**：检查净利润、行业政策、负面事件

#### 买入执行标准（多条件触发）
- 技术趋势、价格形态、启动信号必须同时满足
- 自动计算建议仓位（10%初始，最大20%）
- 自动计算止损价（-7%）和目标价（+15%）

#### 输出功能
- 单个股票详细分析
- 多个股票比较分析
- 风险收益比计算
- JSON格式输出支持

### 2. 每日标准化复盘系统 (`daily_review.py`)
**10分钟高效复盘流程：**

#### 复盘步骤
1. **大盘环境评估**：主要指数20日均线分析
2. **板块强弱分析**：识别强势和弱势板块
3. **自选股管理**：破位删除、走弱减仓、强势持有
4. **观察池优化**：保持5-10只最强势标的

#### 管理功能
- 观察列表的添加、删除、查看
- 自动生成每日复盘报告
- JSON文件持久化存储

### 3. 辅助分析工具
- **`analyze_stock.py`**：8维度股票基本面分析
- **`portfolio.py`**：投资组合创建、添加、查看
- **`dividends.py`**：股息收益率、安全评分分析

## 策略执行要点

### 核心交易原则
1. **趋势优先**：只做上升趋势，禁止抄底
2. **风险分散**：单股≤20%，行业≤30%
3. **严格止损**：下跌7%无条件止损
4. **能力圈内**：只做能理解的交易

### 仓位管理规则
| 市场状态 | 总仓位 | 操作策略 |
|----------|--------|----------|
| 强势市场 | 60-80% | 积极寻找机会 |
| 震荡市场 | 30-50% | 控制仓位，精选个股 |
| 弱势市场 | 0-20%  | 减少操作，观望为主 |

### 风险控制体系
- **事前**：三步筛选排除高风险
- **事中**：7%止损+技术破位止损
- **事后**：每日复盘+持续优化

## 使用示例

### 1. 策略分析示例
```bash
# 分析单个股票
python trading_strategy.py AAPL --detail

# 批量分析比较
python trading_strategy.py AAPL MSFT GOOGL AMZN --compare

# 输出结果示例：
# 🚀 符合买入条件的股票 (2只):
# AAPL    Apple Inc.          $175.50   10.0%   $163.22   $201.83   1:2.1
# MSFT    Microsoft Corp      $415.32   10.0%   $386.25   $477.62   1:2.2
```

### 2. 每日复盘示例
```bash
# 生成复盘报告
python daily_review.py --report

# 报告内容：
# 1. 大盘环境：上证指数健康，深证成指健康
# 2. 板块强弱：科技板块强势，金融板块弱势
# 3. 自选股管理：AAPL持有，TSLA删除（跌破20日线）
# 4. 明日计划：市场健康，可积极寻找机会
```

### 3. 投资组合管理
```bash
# 创建投资组合
python portfolio.py create "稳健投资"

# 添加资产
python portfolio.py add AAPL --quantity 100 --cost 150
python portfolio.py add MSFT --quantity 50 --cost 400

# 查看组合
python portfolio.py show --portfolio "稳健投资"
```

## 注意事项

### 数据限制
- Yahoo Finance数据有15-20分钟延迟
- 盘前盘后数据可能不完整
- 部分A股数据可能受限

### 使用建议
1. **先模拟后实盘**：在模拟账户测试1-3个月
2. **小资金开始**：实盘从小资金开始，熟悉流程
3. **严格执行纪律**：策略成功的关键是纪律执行
4. **定期优化**：根据实际交易记录优化参数
5. **持续学习**：市场变化，策略需要与时俱进

### 风险提示
- **市场风险**：任何策略都无法完全避免市场风险
- **执行风险**：纪律执行不到位是主要风险
- **技术风险**：API变更、网络故障等可能影响
- **过度优化风险**：避免对历史数据的过度拟合
