# Self-Improvement Checklist

Use this checklist periodically to improve your capabilities as an agent.

## After Each Significant Task

- [ ] **Review the work**: Did I accomplish what was requested?
- [ ] **Check for errors**: Are there bugs, typos, or logical errors?
- [ ] **Evaluate completeness**: Is anything missing or incomplete?
- [ ] **Assess quality**: Is the solution elegant, maintainable, and efficient?
- [ ] **Document the process**: What steps did I take? What worked well?
- [ ] **Identify learnings**: What did I learn from this task?
- [ ] **Note improvements**: How could I do this better next time?

## Weekly Review

- [ ] **Scan memory files**: Review `memory/YYYY-MM-DD.md` files from the past week
- [ ] **Extract key insights**: Move important learnings to `MEMORY.md`
- [ ] **Update skills**: Refine skill files based on recent experience
- [ ] **Clean up**: Remove outdated or incorrect information
- [ ] **Plan improvements**: Identify areas for skill development

## Skill Development

- [ ] **Identify gaps**: What tasks am I struggling with?
- [ ] **Research solutions**: How do others solve similar problems?
- [ ] **Create/update skills**: Document new techniques or tools
- [ ] **Test improvements**: Try new approaches on small tasks
- [ ] **Share knowledge**: Update team documentation if applicable

## Memory Management

- [ ] **Organize daily notes**: Ensure `memory/` directory is structured
- [ ] **Curate long-term memory**: Keep `MEMORY.md` focused and valuable
- [ ] **Create cross-references**: Link related information
- [ ] **Prune outdated info**: Remove what's no longer relevant
- [ ] **Backup important data**: Ensure critical knowledge is preserved

## Process Optimization

- [ ] **Identify repetitive tasks**: What do I do frequently?
- [ ] **Automate where possible**: Can scripts or tools help?
- [ ] **Streamline workflows**: Are there unnecessary steps?
- [ ] **Create templates**: For common types of work
- [ ] **Document best practices**: For myself and others

## Meta-Cognition

- [ ] **Reflect on thinking**: How do I approach problems?
- [ ] **Challenge assumptions**: Am I missing alternative perspectives?
- [ ] **Seek feedback**: How do others perceive my work?
- [ ] **Adjust strategies**: What thinking patterns work best?
- [ ] **Cultivate growth mindset**: Embrace challenges as learning opportunities

## Usage Tips

1. **Be honest**: Acknowledge mistakes - they're learning opportunities
2. **Be specific**: "I should test more" vs "I need to add unit tests for API endpoints"
3. **Be actionable**: Identify concrete steps for improvement
4. **Be consistent**: Regular reflection yields compounding benefits
5. **Be patient**: Improvement is gradual, not instantaneous
